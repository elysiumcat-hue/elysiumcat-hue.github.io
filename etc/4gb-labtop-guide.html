납땜 RAM은 사실상 교체 불가입니다. 마이크로 솔더링은 전문 장비·숙련도가 필요하고 메인보드 손상 위험이 크며 비용이 새 노트북 값을 넘기는 경우가 많습니다. 레버리지는 가상메모리 설정과 Windows 10 자체 최적화 쪽에 있습니다.

Priority — 비용 대비 효과 순

효과가 확실한 순서

위에서 아래로 적용하세요. 아래로 갈수록 마지막 카드가 됩니다.

1

SSD 확인 / 교체 // 이미 128GB SSD 보유

HDD였다면 최우선이지만, 이미 SSD이므로 통과. 4GB 환경은 swap(page file)을 자주 타는데 이때 디스크 속도가 곧 체감 속도입니다. SSD라 다음 단계인 가상메모리 설정이 실효성을 가집니다.

2

가상메모리 수동 설정 // 지금 바로 적용

4GB + SSD 조합에서 거의 유일하게 의미 있는 소프트웨어 튜닝. 멈춤(freeze)을 줄여주는 응급 처방입니다. 아래에 단계별 방법과 권장값을 정리했습니다.

3

Windows 10 자체 최적화 // 꾸준히 관리

OS는 Windows 10 그대로 유지하면서 부담을 줄이는 방법입니다. 시작프로그램 비활성화(Ctrl+Shift+Esc → 시작프로그램 탭), 블로트웨어 제거, 디스크 정리로 메모리·디스크 여유를 확보합니다.

잔여물·드라이버 충돌이 누적돼 느려졌다면 OS 클린 설치가 가장 확실한 초기화입니다.

Step-by-step

가상메모리 수동 설정

Windows 10 기준. 경로는 Windows 11과 거의 동일합니다.

먼저 · 가상메모리는 어떻게 동작하나

램이 부족해질 것 같으면 Windows가 한동안 쓰지 않은 데이터를 골라 디스크의 page file로 내보내 램 공간을 확보합니다 (page out). 그렇게 디스크로 보냈던 것 중 다시 필요해지면 램으로 불러들여 사용합니다 (page in).

오가는 단위는 파일이 아니라 page(보통 4KB 크기의 메모리 조각)이며, 어떤 페이지를 내보낼지는 얼마나 오래 안 쓰였는지를 기준으로 정합니다. 그래서 활성 작업은 램에 남고, 한참 방치된 백그라운드 메모리가 먼저 디스크로 밀려납니다.

속도를 빠르게 만드는 게 아니라, 램이 넘칠 때 멈춤을 막아주는 안전망입니다. 디스크는 램보다 수십~수백 배 느리므로 page file을 타는 순간 이미 느려진 상태이고, SSD라서 그 느려짐이 덜할 뿐입니다.

설정 경로 OS: Windows 10
  1. 실행 창 열기 — Win + R → sysdm.cpl 입력 후 Enter
    (또는 시작에서 "고급 시스템 설정 보기" 검색)
  2. 고급 탭 → 성능 영역의 설정 클릭
  3. 성능 옵션 창에서 다시 고급 탭 → 가상 메모리 영역의 변경 클릭
  4. 맨 위 "모든 드라이브의 페이징 파일 크기 자동 관리" 체크 해제
  5. C: 드라이브 선택 → 사용자 지정 크기 선택
  6. 초기 / 최대 크기 입력 → 설정 → 확인
  7. 재부팅 — 적용에 필수
128GB SSD 권장값 단위 · MB
초기 크기
4096 — RAM의 약 1배
최대 크기
8192 — 용량이 작으므로 12GB는 비권장 · 작업/캐시 공간 확보가 우선
왜 8192까지만? 128GB는 용량이 빠듯해서 page file을 너무 키우면 정작 작업·캐시 공간이 부족해집니다. SSD는 빈 공간이 모자라면 쓰기 성능도 떨어지므로, 여유 공간을 최소 15~20GB는 남기는 것을 기준으로 최대값을 잡았습니다.
Storage Sense 켜기 — 설정 → 시스템 → 저장소 → 자동으로 임시 파일·휴지통 정리.
큰 파일은 외부로 옮기기 — 동영상·사진·문서 등 용량 큰 파일은 외장 하드/USB나 클라우드(OneDrive, Google Drive 등)로 옮겨 128GB를 여유 있게 관리하세요.
SSD 수명 걱정은 거의 불필요 — page file 정도의 쓰기로는 최신 SSD의 내구성(TBW)에 유의미한 영향을 주지 않습니다.
적용 확인 — 재부팅 후 Ctrl+Shift+Esc → 성능 탭 → 메모리의 "커밋됨" 상한이 늘었으면 정상 적용.
솔직한 한계 가상메모리 확대는 "부족한 RAM을 디스크로 메우는" 응급 처방입니다. 디스크는 RAM보다 수십~수백 배 느리므로 멈춤은 줄어도 무거운 작업에선 여전히 한계가 있습니다. ReadyBoost는 SSD에선 효과가 없어(Windows가 자동 비활성화) 고려할 필요 없습니다.
